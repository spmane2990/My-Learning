This pipeline addresses both functional and non-functional enterprise requirements for retail transaction processing.

- **Functional Requirements:**
  - **Ingest:** Receive files from many stores via S3 event → handled by the Step Functions entry and `ValidateInput`.
  - **Validation & Quality:** File existence, format, naming, metadata, schema, virus scan, duplicate detection, and business-rule checks → implemented by `validator` and parallel validation Lambdas.
  - **Transformation:** Clean, dedupe, standardize, convert to Parquet and partition → AWS Glue ETL job (`.sync`) does this.
  - **Archival & Traceability:** Copy original input to an archive location for audit and replay.
  - **Analytics & Reporting:** Submit Athena queries, poll for completion, process results via Map state, and generate multiple report formats.
  - **Notification:** Publish success/failure details to SNS with execution/Glue/Athena IDs and report locations.

- **Non‑Functional Requirements:**
  - **Scalability:** Parallel and Map states + serverless services (Lambda, Glue, Athena, S3) scale with load.
  - **Reliability & Resilience:** Retries with exponential backoff, Catch blocks, synchronous Glue integration for deterministic completion, and explicit Fail/Succeed transitions.
  - **Performance & Latency:** Optional “quick” path to skip heavy steps for fast response; concurrency controls (`MaxConcurrency`) and partitioned outputs for efficient reads.
  - **Idempotency & Exactly‑Once Concerns:** Duplicate detection step and archive/copy strategy support idempotent replays.
  - **Observability & Auditability:** Execution context (`$$`), structured result messages, SNS notifications, and archived originals provide audit trail and traceability.
  - **Security & Compliance:** Virus scan, S3-based immutable archives, and centralized notifications help meet security and compliance controls.
  - **Extensibility & Maintainability:** Modular Lambdas, parameterized ASL (`Parameters`, `ResultPath`, intrinsic functions) and clear branching make it easy to add checks or outputs.
  - **Cost Control:** Choice/quick path avoids unnecessary ETL runs; partitioning and Parquet reduce storage and query costs.

- **Where to look in the repo:**
  - State machine definition: state_machine.asl.json — shows orchestration, retries, Catchs, Parallel/Map/Wait/Choice usage.
  - Glue ETL: glue_job.py — shows transformation/partitioning logic.
  - Lambda stubs: lambdas — show validation, verification, Athena submit/check, and report generation.
