# Course Notes: Big Data Applications & Batch Architecture

## 1. Introduction to Big Data Applications

### Characteristics of Batch Processing

Batch processing involves handling a high volume of data in discrete groups (batches) rather than in real time.

- **Schedule/Volume-Driven:** Applications typically run based on a calendar schedule (e.g., hourly, daily) or are triggered automatically once a specific volume of input data accumulates.
- **Discrete Processing:** Data is processed one batch at a time. Each batch contains multiple input records, and the batch size is highly use-case specific (often spanning several gigabytes).
- **High Latency:** There is a significant time delay between when the raw input data becomes available and when the final outputs are derived.
- **Multi-Job Pipelines:** A single big data application may contain multiple interconnected jobs, each scheduled to run at specific intervals.
- **Data Integrity Focus:** Prioritizes data accuracy and completeness. The pipeline will often wait until _all_ expected input data is available, validating data and results at each distinct stage.
- **Failure Resolution:** If a processing error occurs, the standard resolution is to fix the underlying problem and completely reprocess the affected batch data.

### Challenges Building Batch Applications

Handling batch architectures at a "Big Data" scale introduces unique engineering hurdles:

- **Scale and Concurrency:** Storing, transporting, and computing massive volumes of data efficiently requires highly parallelized, concurrent designs. Implementing this concurrency safely is complex.
- **Data Quality issues:** Managing errors, anomalies, and missing data points without breaking the pipeline at scale is difficult.
- **Trust and Integrity:** Ensuring strict end-to-end data integrity across the pipeline so the final business results are completely trustworthy.
- **Resource Management & Costs:** Dynamic and efficient allocation of compute/storage resources to minimize time delays while preventing cost overruns.
- **Monitoring & Troubleshooting:** Sifting through massive data logs to identify, isolate, and resolve pipeline failures in a timely manner.

### Technologies for Batch Big Data Engineering

Building an efficient architecture relies on selecting the right tools across three primary layers:

| Layer               | Purpose                                                          | Key Technology Examples                                                                                                                        |
| ------------------- | ---------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| **Data Processing** | Heavy lifting, scalable compute, and transformations.            | Apache Spark, Apache Flink, Apache Pig, AWS Batch, Apache Storm                                                                                |
| **Data Storage**    | Storing data based on format and access requirements.            | Relational databases, Distributed File Systems (HDFS), Document databases, Columnar databases, Graph databases, Distributed caches             |
| **Data Interfaces** | Acquiring inputs from sources and delivering outputs downstream. | File transfer utilities, custom ETL (Extract, Transform, Load) clients, Data Consuming APIs (push), Query Interfaces (pull), Data Serving APIs |

### Popular Use Cases for Batch Big Data

- **Data Warehousing:** Archiving and analyzing massive volumes of historical data, such as years of transactions and system logs.
- **Global Data Consolidation:** Aggregating data across hundreds of decentralized nodes (e.g., combining daily inventory updates from thousands of individual retail stores into a central data center).
- **Customer 360:** Consolidating user touchpoints across websites, contact centers, social media, and in-store visits to build a unified profile of customer behavior.
- **Domain-Specific Analytics:** Evaluating vertical-specific trends, such as analyzing e-commerce transactions and product page hits to measure performance across different geographies.
- **Machine Learning (ML) Pipelines:** Driving feature engineering by acquiring vast quantities of training data, extracting features, and handling pre-/post-processing for ML inference.
- **Autonomous Automated Workflows:** Supporting status aggregation and background reporting for automated logistics (e.g., tracking an order smoothly from placement to final delivery).

## 2. Architecture Process for Data Engineering

> **Data Engineering Definition:** The methodical process of designing and building big data pipelines that acquire, store, process, and analyze data to derive specific business outcomes.

To successfully architect these systems, engineers follow a structured, step-by-step methodology:

1. **Define the Use Case:** Establish Goals.
   Clearly state the problem to be solved, the expected solution, and overall design goals. This includes identifying functional requirements and defining performance benchmarks.

2. **Study the Requirements:** Analyze Constraints.
   Thoroughly evaluate the required outputs (form and type) and the available inputs (volume, data characteristics, constraints). Define non-functional requirements like system scale, acceptable latency, and target error rates.

3. **Create a Functional Workflow:** Draft the Flowchart.
   Design a baseline flowchart of the pipeline. Focus purely on functional requirements—mapping out how data is acquired, processed, and stored—without worrying about system scale yet.

4. **Scale the Workflow:** Design for Big Data.
   Transform the baseline flowchart into a scalable big data architecture. Evaluate each function independently and inject big data elements so the pipeline can seamlessly handle the expected data volumes.

5. **Select Technologies & Products:** Evaluate Alternatives.
   Establish explicit selection criteria for the technology stack. Evaluate competing products, score them against your criteria, and select the tools that best fit the functions of your application.
